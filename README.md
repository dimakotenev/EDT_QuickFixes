# Quick Fixes for 1C:EDT

Плагин для **1C:EDT**, который добавляет меню **Quick Fixes** — набор команд, исправляющих
типовые замечания стандартных проверок модуля одним кликом.

Идея простая: EDT подсвечивает много однотипных замечаний — методы лежат не в тех областях,
переменные названы не по правилам, остался неиспользуемый код, встречаются устаревшие конструкции.
Руками это править долго и скучно. Плагин делает это за вас: вы открываете модуль, выбираете команду
и сразу получаете результат и короткий отчёт о том, что изменилось.

Каждая операция — **одна правка документа** и **отменяется одним Ctrl+Z**. Имена областей плагин
берёт на языке проекта (русский или английский).

---

## Требования

- **1C:EDT 2025.2+** (собрано и проверено на версии **2025.2.3**).

> Плагин использует внутренние API 1C:EDT, поэтому привязан к версии платформы. На других релизах EDT
> он может не установиться или работать некорректно — в этом случае нужна пересборка под целевую версию
> (см. [Сборка из исходников](#сборка-из-исходников)).

## Установка

### Через "Установить новое ПО"

1. Скачайте файл `release/com.quickfixes.edt-1.0.0-updatesite.zip`.
2. В EDT откройте **Справка → Установить новое ПО…**
3. Нажмите **Добавить… → Архив…** и укажите скачанный zip.
4. Отметьте галочкой **Quick Fixes for 1C:EDT**, нажмите **Далее**, примите условия и **Готово**.
5. Перезапустите EDT, когда он это предложит.

## Что делает

Команды сгруппированы в меню **Quick Fixes** по смыслу:

### Исправление областей
- **Создать стандартные области** — добавляет недостающие стандартные области (`#Область` / `#Region`)
  в правильном для данного типа модуля порядке. Существующие области не трогает и не дублирует.
  Для модуля формы создаёт отдельную область обработчиков под каждую таблицу формы.
- **Расставить области по порядку** — переставляет области верхнего уровня в канонический порядок.
  Если между областями есть «голый» код, операция безопасно отменяется, чтобы ничего не сломать.
- **Перенести методы в корректные области** — берёт методы, помеченные как лежащие не в своей области,
  и переносит их туда, где им положено быть (см. правило ниже). Что перенести не удалось — покажет в отчёте.

### Исправление переменных
- **Имена переменных с заглавной буквы** — приводит имя переменной к правильному виду (первая буква
  заглавная) и переименовывает **все** её вхождения в области видимости. В отчёте — список «было → стало».

### Исправление устаревших конструкций
- **Исправление ошибки `ЭтаФорма`** — `ЭтаФорма.Реквизит` превращает в `Реквизит`, а одиночное
  `ЭтаФорма` — в `ЭтотОбъект` (чтобы не порождать новое замечание).
- **Исправление излишнего обращения к `ЭтотОбъект`** — убирает избыточный префикс: `ЭтотОбъект.Свойство` → `Свойство`.
- **Исправление отсутствующих директив на форме** — вставляет `&НаСервере` над каждым методом, где её не хватает.
- **Исправление доступности с толстого клиента** — оборачивает модуль в
  `#Если Сервер Или ТолстыйКлиентОбычноеПриложение Или ВнешнееСоединение Тогда … #КонецЕсли`
  (один раз, с защитой от повторной обёртки).

### 🗑 Удаление неиспользуемого
- **Удалить неиспользуемые методы и переменные…** — собирает все неиспользуемые методы и локальные
  переменные в окно со списком и галочками (по умолчанию отмечено всё, есть «выделить / снять все»)
  и удаляет то, что вы оставили отмеченным.
- **Удалить пустые строки** — схлопывает несколько подряд идущих пустых строк до одной.

## Куда переносятся методы

Команда «Перенести методы в корректные области» определяет целевую область так:

| Метод | Область |
|---|---|
| Обработчик события (не форма) | `ОбработчикиСобытий` |
| Обработчик формы | `ОбработчикиСобытийФормы` / `…ЭлементовШапкиФормы` / `…ЭлементовТаблицыФормы<Таблица>` / `ОбработчикиКомандФормы` |
| Экспортный метод | `ПрограммныйИнтерфейс` |
| Неэкспортный метод | `СлужебныеПроцедурыИФункции` |
